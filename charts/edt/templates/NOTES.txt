Control plane:
  kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "edt.fullname" . }}-controlplane {{ .Values.controlPlane.service.port }}:{{ .Values.controlPlane.service.port }}

Then open:
  http://localhost:{{ .Values.controlPlane.service.port }}          # UI
  http://localhost:{{ .Values.controlPlane.service.port }}/metrics  # Prometheus

{{- if not .Values.controlPlane.requireAuth }}

Auth is disabled (controlPlane.requireAuth=false). Do not expose the control plane publicly without enabling auth:
  helm upgrade edt ./charts/edt --reuse-values \
    --set controlPlane.requireAuth=true \
    --set controlPlane.adminToken=$(openssl rand -hex 32) \
    --set controlPlane.workerToken=$(openssl rand -hex 32)
{{- else }}

Auth is enabled. Admin token is used by CLI/agents to mutate state; workers use the separate worker-scoped token (RoleWorker).
{{- end }}

Workers: {{ .Values.worker.replicaCount }} replica(s) assigned by the control plane. Submit scenarios:
  curl -X POST http://localhost:{{ .Values.controlPlane.service.port }}/api/v1/scenarios \
    {{- if .Values.controlPlane.adminToken }}
    -H "Authorization: Bearer <admin-token>" \
    {{- end }}
    -H "Content-Type: application/yaml" \
    --data-binary @my-scenario.yaml
